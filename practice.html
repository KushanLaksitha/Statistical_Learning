Regularization, Nonlinear Models, Decision Trees (17+6+3+2+15 = 43 marks)

6 marks

(a) Ridge vs Lasso — Penalty Functions

Ridge Regression (L2 Penalty):
Minimize: Σ(yᵢ − ŷᵢ)² + λΣβⱼ²
• Coefficients toward zero shrink, never exactly zero.
• Variable selection නොකරයි.
• Correlated predictors together shrink (stable).

Lasso Regression (L1 Penalty):
Minimize: Σ(yᵢ − ŷᵢ)² + λΣ|βⱼ|
• Some coefficients exactly zero → automatic variable selection.
• Sparse models produce.
• Correlated predictors: arbitrarily one select (unstable).

Key Difference: Ridge = shrinkage only. Lasso = shrinkage + selection.
6 marks

(b) Linear or Nonlinear in Parameters?

i. Yᵢ = β₀ + β₁e^Xᵢ + εᵢ    ii. Yᵢ = β₀ + β₁Xᵢ^β₂ + εᵢ    iii. Yᵢ = β₀Xᵢ/(β₁ + Xᵢ) + εᵢ

i. Yᵢ = β₀ + β₁e^Xᵢ + εᵢ
→ LINEAR in parameters. β₀, β₁ linear ව appear. e^Xᵢ transformation of X (predictor), not parameter. OLS ගෙන estimate possible.

ii. Yᵢ = β₀ + β₁Xᵢ^β₂ + εᵢ
→ NONLINEAR in parameters. β₂ exponent position ෙදී ඇත. Parameters (β) linear combination නොවේ.

iii. Yᵢ = β₀Xᵢ/(β₁ + Xᵢ) + εᵢ
→ NONLINEAR in parameters. β₁ denominator ෙදී ඇත. Linear algebra ගෙන solve කළ නොහැකිය (Michaelis-Menten model).
3 marks

(c) Nonlinear Models ඇයි?

Linear regression: Straight-line relationship assume. Data හි curves, thresholds, complex shapes ඇතිනම় poor fit.

Examples: Height vs Age (growth curve), Drug dosage vs Effect (saturation), Income vs Education (diminishing returns).

Nonlinear models realistic biological, economic, medical patterns capture.
2 marks

(d) Two Approaches in Nonlinear Regression

1. Modify/Transform Predictors: Polynomial regression (X, X², X³), log transformations — linear framework maintain.

2. Break Predictor Space: Step functions (intervals), splines (piecewise polynomials at knots), local regression (LOESS).
5 marks

(e) Smoothing Splines සහ LOESS

Smoothing splines සහ local regression (LOESS) concept explain. λ parameter role.

Smoothing Splines:
• හැම data point ෙදීම knot place. Overfitting prevent: penalty on curvature add.
• Minimize: Σ(yᵢ − f(xᵢ))² + λ∫[f″(x)]² dx
• λ large → smoother curve (high bias, low variance).
• λ small → wiggly curve (low bias, high variance).
• GCV (Generalized Cross-Validation) ගෙන optimal λ select.

LOESS/LOWESS (Local Regression):
• Global model fit කරන්නේ නෑ — locally around each point weighted regression.
• Span parameter (s): proportion of data used per local fit.
• Small span → flexible, overfit risk. Large span → smooth, underfit risk.
• Weights: tricube weight function — nearer points get more weight.

R
R
# Smoothing Spline
ss_fit <- smooth.spline(x, y, cv = TRUE)
plot(x, y, pch=16); lines(ss_fit, col="red", lwd=2)

# LOESS
lo_fit <- loess(y ~ x, span = 0.5)
x_grid <- seq(min(x), max(x), length.out=100)
lines(x_grid, predict(lo_fit, x_grid), col="blue", lwd=2)

4 marks

(g) Elastic Net Regression

Elastic Net ෙදී α parameter role. Ridge සහ Lasso සමඟ compare.

Elastic Net Penalty:
Minimize: Σ(yᵢ − ŷᵢ)² + λ[α Σ|βⱼ| + (1−α) Σβⱼ²]

• α = 1 → Lasso (L1 only). α = 0 → Ridge (L2 only). 0 < α < 1 → Elastic Net.
• Correlated predictors → groups select (Ridge component).
• Sparse model produce (Lasso component).
• Lasso problem: correlated predictors ෙදී arbitrarily one select → Elastic Net ෙදී stable.

R
R

Logistic Regression – Classification (30 marks)

File: Q3-YourRegNumber.Rmd
4 marks

(a) Why Not Linear Regression for Classification?

Binary outcome (0/1) predict කිරීමට linear regression ගත නොහැකි ඇයි? Logistic regression solution.

Linear Regression Problems for Classification:
1. Range violation: Predictions outside [0,1] → probabilities invalid (-0.3 or 1.8 probability meaningless).
2. Heteroscedasticity: Binary Y → non-constant error variance.
3. Poor interpretability: Predicted values as probabilities inappropriate.

Logistic Regression Solution:
• Logistic (sigmoid) function: P(Y=1|X) = 1/(1 + e^(−z)), z = β₀ + β₁X
• Output always ∈ [0,1] → valid probability.
• S-shaped curve → classification natural.
• MLE (Maximum Likelihood Estimation) ගෙන parameters estimate.
5 marks

• Balance $1 increase → default odds 1.006 times (0.6%) increase.
• β̂ > 0 → positive relationship. β̂ < 0 → protective effect.

Cross-Validation Methods (30 marks)

File: Q4-YourRegNumber.Rmd
5 marks

(a) Why Cross-Validation?

Training error vs test error problem. CV ගෙන solve කරන ආකාරය.

Training Error Problem:
• Model train data ෙදී fit → training error always decreases with complexity.
• Overfitting: training error low, test error high → poor generalization.
• Test set separate රඛිය නොහැකිනම் error estimate impossible.

CV Solution:
• Training data ෙදීම split → train + validate.
• Multiple rounds → stable test error estimate.
• Model selection (degree, k, λ) ෙදී use.
• No additional test data needed.

Methods: Validation Set, LOOCV, K-Fold CV (most common), Stratified CV.
6 marks

(b) K-Fold CV — Polynomial Degree Selection

data.set.1.csv ගෙන 10-fold CV. Degrees 1–25 test. Best degree identify.

R
R
df <- read.csv("data.set.1.csv", header = TRUE)
set.seed(123)
n <- nrow(df)
K <- 10
folds <- sample(rep(1:K, length.out = n))

cv_error <- function(degree) {
  mse <- numeric(K)
  for (k in 1:K) {
    train <- df[folds != k, ]
    test  <- df[folds == k, ]
    fit   <- lm(y ~ poly(x, degree, raw = TRUE), data = train)
    pred  <- predict(fit, newdata = test)
    mse[k] <- mean((test$y - pred)^2)
  }
  mean(mse)
}

degrees <- 1:25
cv_results <- sapply(degrees, cv_error)
best_degree <- degrees[which.min(cv_results)]
cat("Best polynomial degree:", best_degree)

plot(degrees, cv_results, type="b", pch=16, col="steelblue",
     xlab="Polynomial Degree", ylab="10-Fold CV MSE")
abline(v=best_degree, col="red", lty=2)
Interpretation: CV MSE plot U-shape show. Minimum point = optimal degree. Too low degree → underfitting (high bias). Too high → overfitting (high variance).
4 marks

(c) LOOCV vs K-Fold CV

Leave-One-Out CV සහ K-Fold CV compare. Bias-Variance tradeoff.

Feature LOOCV K-Fold (K=10)
Training size n−1 (very large) ≈ 0.9n
Bias Very low Slightly higher
Variance High (correlated folds) Lower (better)
Computation n models (slow) K models (fast)
Practical choice Small n K=5 or K=10 preferred

Recommendation: K=10 fold CV — best bias-variance balance. Computationally feasible.
5 marks

(d) Bootstrap — When Ideal?

Bootstrap ideal situations 5 explain. When should use bootstrap?

Bootstrap Ideal ෙදී:

1. No closed-form SE: Median, quantiles, trimmed mean — theoretical formula unavailable. Bootstrap computational approach provide.

2. Assumption violations: Non-normal data, heteroscedasticity. Bootstrap requires only i.i.d. observations.

3. Small samples: Asymptotic approximations poor. Bootstrap still reliable.

4. Complex estimators: AUC, robust regression, ML metrics — exact distributions derive කළ නොහැකිය. Resample + recalculate.

5. CI without parametric assumptions: Percentile CI naturally skewness reflect — symmetric parametric CI than better.

Rule: Statistic compute හැකිනම් but variability නොදනිනම් → Bootstrap ideal.
5 marks

(e) Bootstrap Confidence Interval — Median

Exponential data ෙදී bootstrap ගෙන median 95% CI compute.

R
R
set.seed(123)
x <- rexp(50, rate = 0.5)
B <- 2000
n <- length(x)
boot_median <- numeric(B)

for (b in 1:B) {
  xb <- x[sample(1:n, size=n, replace=TRUE)]
  boot_median[b] <- median(xb)
}

ci_median <- quantile(boot_median, c(0.025, 0.975))
cat("Sample Median:", median(x), "\n")
cat("Bootstrap SE:", sd(boot_median), "\n")
cat("95% CI:", ci_median)
5 marks

(f) Bootstrap Hypothesis Test — Median H₀: m = 5

Sample data: c(6.2, 5.1, 4.8, 5.5, 6.0, 4.9, 5.3, 6.1, 5.7, 4.6). H₀: median = 5 test.

R
R
set.seed(123)
x <- c(6.2, 5.1, 4.8, 5.5, 6.0, 4.9, 5.3, 6.1, 5.7, 4.6)
m0 <- 5; n <- length(x); B <- 5000

boot_meds <- numeric(B)
for (b in 1:B)
  boot_meds[b] <- median(sample(x, n, replace=TRUE))

# Two-sided p-value
obs_med <- median(x)
p_val <- mean(abs(boot_meds - obs_med) >= abs(obs_med - m0))
cat("Observed Median:", obs_med, "\n")
cat("Bootstrap p-value:", p_val)
Decision: p-value < 0.05 → H₀ reject (median ≠ 5 significant). p-value > 0.05 → fail to reject (insufficient evidence against H₀: median = 5).
